JS总结(1)

错误处理机制

1.错误类型

  • SyntaxError 解析代码时发生的语法错误
  • ReferenceError 引用一个不存在的变量时发生的错误
  • RangeError 溢出错误
  • TypeError 变量或参数类型错误
  • URIError URI参数不正确
  • EvalError eval函数错误执行(ES5已经不再出现)

    2.throw语句

    中断程序执行，并抛出一个意外或错误

    3.try…catch

捕捉错误，并立即执行catch中的函数
同时，该结构后面可以加入final代码块来执行不管是否发生错误，都会执行的代码块

函数

函数表示一段可以反复调用的代码块。

  • 函数名提升: 函数和变量名一样，在声明事会像变量声明一样，被提升到代码(外层函数)头部，但如果将函数视为变量值，则函数名没有提升到代码头部，需要事先声明，否则报错。

5.参数

  • 参数可以省略,但是无法只省略靠前的参数，而保留靠后的参数，如果一定要这样做的话，只能显示地给前面的参数传入undefined，如果存在同名参数，则取最后出现的参数。

  • 闭包 ：函数外部无法读取函数内部的变量，闭包可以用来实现函数内部和外部的连接，同时，闭包的诞生环境一直存在，所以这些变量一直保存在内存中。

对象

1.概述

对象是JavaScript的核心概念，也是最重要的数据类型，是一种无序的数据集合，由若干键值对构成。键名又可以成为属性，属性的值如果是函数，则成为方法

1.1生成方法

  • 直接使用大括号{}生成
  • 使用new命令生成一个object对象实例
  • 使用Object.create生成

对象的引用：对象引用了对象的内存地址，修改其中一个变量会影响到其他变量(对于原始数据类型则是传值引用，也就是值的拷贝，一个变量的变化不会影响其他变量的值)
为了避免歧义，规定如果行首是{则解释为语句，如果要表达为对象可以加圆括号

2.属性

  • 属性的读取与赋值：
    • 点运算符
    • 中括号运算符
  • 查看对象的所有属性: Object.keys(obj)
  • delete命令，用来删除对象的属性,只能删除本身的属性，无法删除继承的属性，同时无法删除Object.defineProperty创建的属性
  • for..in用于遍历对象所有可遍历的属性，跳过不可遍历属性，不仅遍历对象自身的属性，还遍历继承的属性。

其他（tips）

  • 避免全局变量，如果不得不使用，用大写字母表示变量名，如 UPPER_CASE
  • 函数都在使用之前定义，变量的声明放在函数头部
  • 不要使用“相等”（==）运算符，只使用“严格相等”（===）运算符。
  • ++,--使用+=,-=代替
  • 避免使用switch...case结构，而使用对象结果替代。
