  • this的指向是在函数被调用的时候确定的。
  • 函数执行过程中，this的指向不会发生改变
  • 在一个函数上下文中，this由调用者提供，由调用函数来确定。如果调用函数被某一个对象所拥有，则该函数被调用时，内部的this指向该对象（如var obj=function(){}，this的指向就是obj），如果函数独立调用，那么该函数内部的this指向undefined，在非严格模式下，当this指向undefined时，会自动指向全局对象。

  2. 跨域方法

  • 为什么需要跨域

    浏览器的同源策略,即浏览器不允许跨域调用其他页面的对象,只有当协议，域名，端口相同时才能算是同一个域名下，否则都无法调用，为了实现跨域调用需要使用跨域方法

  • 跨全域：
    • jsonp: 利用的原理是，页面上调用js文件不受浏览器同源策略的影响。即前端模拟了一种请求js文件的方法，拿到的结果是script标签的内容，可以在浏览器中运行，从而实现跨域
    • cors：跨域资源共享。即允许浏览器向跨源服务器发出xhr请求，需要的条件是前端和服务器需要同时支持才能生效，前端浏览器需要支持xhr，服务器需要设置access-control-allow-origin,该值需要和请求头中的origin一致才能生效
    • 代理模式：即在跨域服务器和前端之间加入一个中间件，让其代理前端请求。
  • 其他页面通信方式：
    • location.hash 通过哈希值来进行数据的传递
    • window.name
    • postMessage
    • document.domain
